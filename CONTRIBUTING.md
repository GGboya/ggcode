# 贡献指南（Contributing Guide）

感谢你对 GGCode 的关注和贡献！我们欢迎任何形式的贡献，包括但不限于：代码、文档、测试、建议和反馈。

## 贡献流程

1. Fork 本仓库
2. 新建分支（建议以 feature/xxx 或 fix/xxx 命名）
3. 提交更改（请写清晰的 commit message）
4. 推送分支到你的 Fork
5. 在 GitHub 上创建 Pull Request
6. 等待项目维护者 Review 和合并

## Issue 提交规范
- 请尽量描述清楚问题的复现步骤、期望行为和实际行为
- 如有报错信息请附上完整日志
- 建议配图或录屏说明问题

## Pull Request 规范
- 保证代码通过本地测试
- 遵循项目的代码风格（建议使用 gofmt/goimports）
- 每个 PR 只做一件事，避免混合提交
- 如涉及 UI/UX 变更请附截图

## 代码规范
- Go 代码建议使用 gofmt/goimports 格式化
- 变量/函数/文件命名需语义清晰
- 推荐英文注释，关键逻辑需有注释说明

## 社区行为
- 请遵守 [行为准则](./CODE_OF_CONDUCT.md)
- 尊重他人，友善沟通

## 联系我们
如有疑问或建议，欢迎通过 Issue 或 Discussions 与我们联系。
